## Package Overview

R simulation package for Management Strategy Evaluation (MSE) of Pacific salmon stocks. Accompanies Ohlberger, Schindler & Staton (2025), *Fish and Fisheries* 26:113–130.

Four exported functions: `run_scenarios()`, `make_figures()`, `param_configs` (dataset), `convert_legacy_to_ssi()`.

## Key Commands

```r
devtools::test()        # Run test suite
devtools::check()       # Full R CMD CHECK
devtools::document()    # Regenerate NAMESPACE and man/*.Rd from roxygen2
pkgdown::build_site()   # Rebuild documentation website
```

**Run `devtools::document()` before every commit** — NAMESPACE and `man/*.Rd` are generated by roxygen2 and must stay in sync with source.

No renv: dependencies are declared in DESCRIPTION only (no lockfile).

## Code Style

- 2-space indentation
- Internal helpers use a dot prefix: `.calc_ricker()`, `.ensure_outdir()`
- Exported functions are snake_case with no dot prefix
- Roxygen2 with markdown enabled (`Roxygen: list(markdown = TRUE)`)

## Testing

Edition: testthat 3 (`Config/testthat/edition: 3`).

- Tests always run with `parallel = FALSE` for reproducibility
- The integration test in `test-integration.R` calls `run_scenarios()` end-to-end and writes diagnostic plots to `outputs/test_runs/<timestamp>/` (gitignored)
- Never commit contents of `outputs/` or `figures/` — both are gitignored

## Architecture Notes

- **Two parameter profiles**: `param_configs$Ohlberger` and `param_configs$Kuskokwim`
- **Parallel execution**: uses `future::multisession`; automatically sets `OMP_NUM_THREADS=1` to prevent BLAS oversubscription — never enable parallel in tests
- **Silent corrections**: model fallbacks and bias corrections are logged to CSV in `outputs/logs/`; check these during QA
- **Legacy bridge**: `convert_legacy_to_ssi()` converts outputs from the original `janohlberger/SizeShiftsImplications` repo to the current `ssi_run` structure
